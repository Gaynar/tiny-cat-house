# Game Design Document - Hector's Adventure

> Status: first rewrite from `hector-roguelike-design-questions.md`.
>
> This document replaces the Kitty Tower Idle direction as the main design source for new work. The old GDD can remain as historical reference for project stack notes, asset conventions, and lessons learned, but game systems should follow this document.

---

## 1. High Concept

Hector's Adventure is a cozy-by-day, spooky-by-night roguelike about Hector, a sweet house cat who prepares at home during the day and explores dangerous alleys after dark.

By day, Hector lives in a fixed terraced house. Rooms passively generate resources, provide small preparations, and improve Hector through permanent upgrades. The house is shown as a side-cutaway dollhouse, with Hector appearing in multiple rooms at once to create the illusion that a quick day contains many tiny cat routines.

By night, Hector ventures into the Back Alley on a branching roguelike map. He fights cartoon scuffles, visits events, rests, shops, challenges elites, and faces a boss every night. If he wins, he returns home with resources and progress. If he loses, he still returns home, because cats have more than one life.

The fantasy is:

> Spend the day being an adorable house cat. Spend the night being a tiny alley hero.

---

## 2. Core Design Pillars

### 2.1 Cozy Home, Spooky Alley

The day phase should feel cozy and comedic: naps, grooming, eating, playing, and other ordinary cat routines. The night phase should feel spooky and comedic: bins, shadows, rats, birds, dogs, and rival cats become cat-sized threats.

The game should not use realistic violence. Combat is cartoon scuffling.

### 2.2 Hector First

Hector is always the main character and always in the party. Later versions may support a party of up to three cats and switching party members, but the MVP has only Hector.

For now, Hector is a Fighter. Later, each cat can act as their own class, and Hector can become special by combining three classes at once.

### 2.3 Runs Reset, Home Remains

Night runs are temporary. Hector starts a run at level 1. During the run he gains XP, levels up, buys temporary abilities, finds temporary items, and fights toward the boss.

On defeat, Hector returns home. He loses the run level, run abilities, and run items, but keeps any obtained home resources. House upgrades are permanent.

### 2.4 Small Permanent Progression

Permanent progression should come mainly from house upgrades, not permanent combat talents. Upgrades give small flat stat bonuses, start-of-run supplies, or quality-of-life benefits.

### 2.5 Readable Tactical Combat

Combat should be simple enough for a first MVP but readable enough to scale. Enemies should show intent before they act so the player can make meaningful choices with Attack, Ability, Item, Defend, and Flee.

---

## 3. Core Decisions

- Working title: Hector's Adventure
- Tone: cozy/comedic during the day, spooky/comedic at night
- Main character summary: Hector is sweet and cute at home, tough in the alley
- Core day loop: short real-time day timer, passive room preparation, spend resources on room upgrades
- Core night loop: choose nodes on a generated alley map, fight and survive until the boss
- Combat style: turn-based cartoon scuffles with enemy intent
- Main permanent progression: room upgrades and house benefits
- Main run-specific progression: XP, level ups, temporary abilities, temporary items
- Defeat consequence: return home, reset run level/items/abilities, keep gathered resources
- First MVP scope: Hector only, fixed house, Back Alley, one boss, basic combat, basic map, basic upgrades

---

## 4. Target Player Experience

### 4.1 Quick Session

The player opens the game, watches a short day pass, checks what the house prepared, starts a night run, and makes a few meaningful node and combat choices.

### 4.2 Run Session

The player navigates a visible node map, fights 3 to 6 turn battles, picks temporary abilities after level ups, uses items, rests when needed, and tries to beat the boss.

### 4.3 Upgrade Session

The player returns home with Fishbones and Canned Tuna, upgrades rooms, improves Hector's next run, and tries again with a slightly better house.

---

## 5. Resources

The MVP should use two resources.

### 5.1 Fishbones

Fishbones are the main permanent currency.

Generated by: house rooms, night rewards, events, elite nodes, boss wins.

Used for: room upgrades, future shop purchases, and permanent preparation improvements.

### 5.2 Canned Tuna

Canned Tuna is food and healing preparation.

Generated by: Kitchen upgrades and possibly day phase room output.

Used for: healing items during night runs, rest/event choices, and later cooking-style upgrades.

### 5.3 Later Resources

Do not add more resources in the MVP. Add a third resource only when a system clearly needs it. Possible later resources include Scrap for tinkering or Curiosity for discoveries.

---

## 6. Day Phase

### 6.1 Day Timing

One day is a short real-time timer. MVP target: 30 seconds.

Offline progress does not exist. The game only progresses while open.

### 6.2 House Presentation

The house is fixed and shown as a side-cutaway dollhouse.

Layout:

| Floor | Room |
|---|---|
| 3 | Attic |
| 2 | Bedroom |
| 2 | Storage/Washing Room |
| 1 | Living Room |
| 1 | Kitchen |

Rooms do not visually change when upgraded in the MVP. Visual upgrade states can come later.

### 6.3 Room Activities

Every unlocked room should show Hector doing something. Multiple visible Hectors are acceptable because the day passes quickly and creates the feeling of a busy cat routine.

Room roles:

| Room | Activity | Gameplay role |
|---|---|---|
| Living Room | Play | Small Attack or Speed preparation |
| Kitchen | Eat | Generates Canned Tuna or start-of-run healing |
| Bedroom | Nap | Max HP or starting HP preparation |
| Storage/Washing Room | Litterbox | Small Luck bonus or event outcome bonus |
| Attic | Storage | Later prepared-item storage |

### 6.4 Room Upgrades

Room upgrades are permanent and tiered. MVP upgrades should be mechanically small and easy to understand.

Suggested MVP effects:

| Room | Tier 1 | Tier 2 | Tier 3 |
|---|---|---|---|
| Living Room | +1 Attack | +1 Speed | start run with Inspired for 1 battle |
| Kitchen | +1 Canned Tuna per day | start run with 1 Tuna Snack | start run with 2 Tuna Snacks |
| Bedroom | +5 Max HP | +10 Max HP | heal to full before each night |
| Storage/Washing Room | +1 Luck | better event reward chance | reduce Flee penalty |
| Attic | unlock item storage later | hold 1 prepared item later | hold 2 prepared items later |

For MVP, only implement the first tier if needed to keep scope small.

---

## 7. Calendar And Day/Night

The calendar is mostly flavor and progress tracking. It helps the player see how many days they have spent trying to finish the game.

Weekdays, seasons, weather, date-specific events, and story gates are not part of the MVP.

Hector can go out every night.

---

## 8. Night Map

### 8.1 Map Rules

The alley map is generated fresh every night.

MVP structure:

- One zone: Back Alley
- Around 8 nodes before the boss
- All node types are visible in advance
- Hector can abandon a run and return home early

### 8.2 Node Types

MVP node types:

| Node | Purpose |
|---|---|
| Combat | normal battle and XP |
| Event | choice or short outcome |
| Rest | heal or recover MP |
| Shop | placeholder for later, can be disabled in first implementation |
| Elite | harder combat with better rewards |
| Boss | final fight of the night |

Shops are in the design but can be deferred until items and shop identity are clearer.

---

## 9. Combat

### 9.1 Battle Shape

Combat is turn-based. Hector can fight 1 to 5 enemies.

Normal battles should usually last 3 to 6 turns. Boss battles should last around 10 turns.

### 9.2 Combat Stats

Hector and enemies use:

- HP
- MP
- Attack
- Defense
- Speed
- Luck

### 9.3 Player Actions

MVP actions:

| Action | Effect |
|---|---|
| Attack | basic damage to one enemy |
| Ability | spend MP for active or passive class skills |
| Item | use a temporary run item, such as Canned Tuna |
| Defend | reduce incoming damage for the turn |
| Flee | attempt to leave non-boss combat |

### 9.4 Enemy Intent

Enemies should show intent before acting. This is recommended for MVP because it makes combat clearer and gives Defend, Item, and target choice real value.

Example intents:

- Attack
- Strong Attack
- Defend
- Apply Wet
- Apply Hungry
- Apply Spooked

### 9.5 Status Effects

MVP statuses:

| Status | Suggested effect |
|---|---|
| Wet | reduce Speed or increase incoming damage slightly |
| Hungry | reduce Attack or slowly lose HP |
| Spooked | chance to miss or inability to use some abilities |

Keep status rules simple until combat feel is proven.

---

## 10. Hector Progression

### 10.1 Run Level

XP is not permanent. Hector starts each night run at level 1.

After reaching XP thresholds, Hector levels up. Leveling grants points or choices used to unlock temporary abilities for the current selected class.

### 10.2 Fighter Class

MVP Hector is a Fighter with up to 10 unlockable abilities in a simple tree. Abilities can be active or passive.

Example ability tree:

| Ability | Type | Example effect |
|---|---|---|
| Sharp Swipe | Active | MP attack for extra damage |
| Guarded Paws | Passive | Defend blocks more damage |
| Quick Scratch | Active | low damage, acts early |
| Nine-Life Nerve | Passive | survive once at 1 HP per run |
| Alley Pounce | Active | bonus damage to wounded enemy |
| Thick Fur | Passive | +1 Defense for this run |
| Focused Stare | Active | apply Spooked |
| Second Wind | Passive | recover small HP after battle |
| Claw Combo | Passive | basic attacks sometimes hit twice |
| Tiny Champion | Active | expensive high damage attack |

These are starting ideas, not locked balance.

### 10.3 Later Party Progression

Later, the game may support a maximum party of three cats. Each cat can be its own class. Hector can become unique by combining up to three class identities.

Do not build party switching in the MVP.

---

## 11. Items And Shops

Items and shops are later systems. The MVP can include only simple Canned Tuna healing items if needed for combat testing.

Do not design a full shop economy until the core day/night loop is playable.

---

## 12. Bosses

There is a boss every night.

Bosses do not gate story progression in the MVP. Later, bosses can unlock new alley zones, but not yet.

Bosses should be repeatable with scaling difficulty.

MVP boss rule:

> A boss can start as a regular enemy archetype with 3x health and 1.5x damage.

First boss recommendation: The Bin Baron.

The Bin Baron is a huge grumpy alley rat or rival cat guarding the best trash bin in the Back Alley. MVP implementation can reuse the Rat or Rival Cat enemy profile, with boss stat multipliers.

---

## 13. Enemies

MVP enemy families:

- Rats
- Birds
- Dogs
- Rival cats

Later enemies can become more unrealistic as the game goes deeper into the alley world.

Suggested first enemies:

| Enemy | Role |
|---|---|
| Alley Rat | basic attacker |
| Bold Pigeon | evasive or Speed-focused enemy |
| Tiny Dog | high Attack, obvious intent |
| Rival Tabby | balanced enemy with Defend |
| Bin Baron | first boss |

---

## 14. Run Variety

Each night should feel different through a combination of:

- Fresh generated map
- Different visible node paths
- Different enemy group sizes
- Random events
- Different level-up ability choices
- Temporary items and rewards
- Optional elite risk/reward

Do not rely on permanent XP for variety.

---

## 15. Existing Content And Migration

The previous Kitty Tower Idle content is not part of the new direction.

Remove or replace:

- Miso, Bean, and Mochi
- Cat roster assignment
- Relationship systems
- Furniture identity systems
- Offline progress
- Coins and Comfort economy
- Tower expansion as the main game
- Idle production formulas based on cat placement

Potentially keep or adapt:

- React + Vite project structure
- localStorage save/load approach
- Lucide icons
- Pixel-art asset conventions
- Cutaway room presentation idea
- Resource bar pattern
- Pure store/action/test style where useful

---

## 16. MVP Scope

The first playable version should include:

- One playable cat: Hector
- One fixed terraced house with five rooms
- 30-second day timer
- Passive room preparation
- Fishbones and Canned Tuna
- Permanent room upgrades
- One generated Back Alley map per night
- Visible node types
- Combat, Event, Rest, Elite, and Boss nodes
- Shop node placeholder or disabled shop node
- Turn-based combat with Attack, Ability, Item, Defend, Flee
- MP-based abilities
- Run XP and temporary ability unlocks
- One Fighter ability tree
- One boss: The Bin Baron
- Around 8 to 12 total night encounters/events
- Defeat returns Hector home and resets run progress

Cut from MVP:

- Permanent XP talents
- Party members
- Switching cats
- Offline progress
- Full shop system
- Multiple zones
- Seasons, weather, story gates
- Visual room upgrade changes

---

## 17. Technical Direction

Use the existing app stack unless there is a strong reason to change:

- React + Vite
- JavaScript
- localStorage persistence
- Lucide React icons
- CSS-first UI
- Pixel-art rendering for game assets with `.pixel-art`

Recommended save key for the new game:

```js
hectorsAdventure_save
```

The current `kittyTowerIdle_save` key should not be reused for the new game, so old saves do not create strange migration bugs during the pivot.

## 18. Open Questions

These do not block the MVP:

- Should the Back Alley be cat-sized only, or become unrealistic earlier?
- What exact stat numbers should Hector and enemies use?
- Should Fishbones drop mostly from night runs, house rooms, or both?
- How generous should Canned Tuna healing be?
- Should Shop nodes be hidden until the shop system exists?
- What is the long-term win condition if there is no story yet?
